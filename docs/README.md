# 📄 Documentation Guide

This folder contains detailed documentation for the **Blogging Platform Full Stack Application**.
It is intended to provide a clear understanding of the system’s backend, database design, setup process, and deployment.

---

## 📚 Documentation Files

### 🔹 API.md

Contains all backend API endpoints with request and response details.

* Authentication APIs (Register, Login)
* Post APIs (Create, Read, Update, Delete)
* Comment APIs (optional feature)

---

### 🔹 DATABASE.md

Describes the database structure used in the project.

* Tables: `user`, `post`, `comment`
* Relationships (Foreign Keys)
* Schema explanation

---

### 🔹 SETUP.md

Step-by-step guide to run the project locally.

* Backend setup (Spring Boot)
* Frontend setup (React)
* Database configuration (MySQL)

---

### 🔹 DEPLOYMENT.md

Instructions to deploy the application.

* Frontend deployment (Vercel)
* Backend deployment (Render)
* Database hosting (Railway)

---

## 🧠 System Overview

The application follows a **full-stack architecture**:

* **Frontend** → React (User Interface)
* **Backend** → Spring Boot (REST APIs)
* **Database** → MySQL (Data Storage)
* **Authentication** → JWT-based security

This documentation helps to:

* Learn API structure and database design
* Support project evaluation and demonstration
